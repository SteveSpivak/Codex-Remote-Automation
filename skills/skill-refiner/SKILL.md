# Skill Refiner

## Purpose

Upgrade existing skills without turning every revision into a full rewrite.

## Process

1. Read the current SKILL.md and any bundled resources only as needed.
2. Check trigger quality, overlap, deliverables, and packaging.
3. Remove text that does not change behavior.
4. Tighten the workflow and output contract.
5. Regenerate or validate agents metadata if the skill meaning changed.

## Hard Rules

- Preserve the skill's core job unless the user asks for a repositioning.
- Fix overlap by sharpening triggers before adding more content.
- Prefer deleting weak text over rephrasing it.
- Keep packaging minimal and validation explicit.

## Deliverables

- Revised skill direction
- Overlap notes
- Packaging corrections
- Validation checklist

## Output Format

- Current weakness
- Recommended refinements
- Packaging changes
- Validation steps
