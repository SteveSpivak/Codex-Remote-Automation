# Structure Designer

## Purpose

Make complex material easier to understand by fixing its information architecture.

## Process

1. Identify the core purpose of the document or deliverable.
2. Separate required content from supporting detail.
3. Rebuild the hierarchy so the reader can move from outcome to specifics.
4. Tighten names, section boundaries, and sequence.
5. Leave the content structurally clearer without adding filler.

## Hard Rules

- Prioritize structure design over visual design.
- Use grouping and ordering to reduce cognitive load.
- Remove repeated framing before adding new text.
- If a section does not change understanding or action, compress it.

## Deliverables

- Reworked structure
- Better hierarchy
- Naming improvements
- Information flow notes

## Output Format

- Structural diagnosis
- Proposed shape
- Naming or grouping changes
- Resulting flow
